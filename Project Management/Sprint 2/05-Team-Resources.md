## Processes & Methods

### Agile / Scrum (Continued)

- **Daily standups** to coordinate frontend-backend integration points.
- **Mid-sprint UI review** with stakeholders to validate the dashboard and key workflows.
- **Sprint review** demo showcasing the complete user interface end-to-end.
- **Sprint retrospective** with focus on frontend-backend communication lessons learned.

### Design Process

- **Wireframes** created for key pages (Dashboard, Order Detail) before implementation.
- **Component library audit** – Syncfusion components were evaluated and selected for grid, form, and chart widgets.
- **Iterative feedback** – Frontend developers paired with the backend developer to resolve API shape mismatches in real time.

### Development Practices

- **Feature branches** per view area (e.g., `feature/dashboard`, `feature/purchase-views`).
- **Cross-functional pairing** – Frontend developers worked alongside the backend developer to test API integration.
- **Browser testing** – Views verified in Chrome, Edge, and Firefox.
