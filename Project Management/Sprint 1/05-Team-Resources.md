## Processes & Methods

### Agile / Scrum

- **Daily standups** (15 min) to synchronize progress and remove blockers.
- **Sprint planning** session at sprint start to size and assign user stories.
- **Sprint review** demo at sprint end to showcase the working backend API.
- **Sprint retrospective** to identify process improvements for Sprint 2.

### Development Practices

- **Git branching:** Feature branches off `master` with pull-request reviews.
- **Code reviews:** All pull requests required at least one approving review before merge.
- **Continuous Integration:** Automated build triggered on every push to validate compilation.
